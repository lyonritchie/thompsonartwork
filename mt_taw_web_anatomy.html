ThompsonArtwork.com is a static website — it does not use a database. Instead data is stored in flat text files throughout the git repository. GitHub Pages automatically re-compiles this repo to html after each edit (commit) or push from a local working copy. To streamline maintenance and publishing, NetlifyCMS is connected as a front-end available at https://thompsonartwork.com/admin.

Requirements

  • A GitHub user account

  • Write privileges to the site repository provided by invite from Lyon Ritchie

  • Images named word-word-another.jpg, e.g. using the dash as a separator. Include corresponding thumbnails:

    • Size: 250x250 px

    • Name: ends with _tn.ext, e.g. word-word-another_tn.ext

Content Editing

Hints are included with notable fields in the NetlifyCMS editor. Fields for editable content types are also listed below.

Blog

Fields

  • Title

  • Published

  • Publish Date. Required.

  • Body - Use the Rich Text editor to add images to your post.

  • Post Thumbnail - Used in the blog post teaser/preview.

  • Description - Search engine optimization (SEO): Will be used in the site meta-description tag.

  • Topics - Be sure to match the topic name used in Topics (lowercase, dashes, etc.)

You can upload images using the Rich Text editor in NetlifyCMS. Switch back to Markdown to apply styling.

To bulk upload images to a folder:

  1. Navigate to the folder on GitHub

  2. Click the Upload files button on the right, above the file list (you must be logged in)

  3. Drag and drop multiple files

  4. Click Commit changes

Body Field Content

Figure 3. NetlifyCMS Rich Text editor, button bar.
1 Headings within blog posts should begin at H2. Insert using the Rich Text editor (H), or ## in Markdown.
2 Images can be uploaded and inserted with the image component.

Bootstrap classes can be used for formatting as shown below. Bootstrap classnames appear between brackets { …​ } and immediately follow the image tag.

Image styling using Bootstrap classes (Markdown view):
![Molly Thompson](/uploads/molly-2020.png){: (1)
 .w-50 (2)
 .ml-3 (3)
 .img-thumbnail (4)
 .float-right (5)
}
1 (Colon required)
2 Scale the image to 50%
3 Add left margin of 3 units
4 Alter image appearance, and
5 Align the image to the right of the content area.

Gallery Items

Fields

  • Title - Appears below image. Required.

  • Description - Used only for SEO + Social Sharing (via Open Graph).

  • Image - Images uploaded to /uploads/[galleryname]/. Required.

  • Image Thumbnail - Images uploaded to /uploads/[galleryname]/thumbnail/. Required if not uploading them manually via GitHub.

  • Published

  • Publish Date - Affects sort order in gallery. Required.

  • Tags - Options: "cover", "featured". Show image on home page (featured) or as the gallery cover image.

  • Body - Optional content will appear below the image.

Gallery images are stored separate from the default Media location.

Archives and Topics

Archive and Topic entries must exist to provide lists of blog posts organized by year or by topic.

Fields for Archives

  • Year - in YYYY format

  • Published

  • Body

Fields for Topics

  • Topic - Lowercase, use dash for space between words.

  • Published

  • Body

Pages

Fields

  • Order - Controls the position within the main navigation

  • Body

  • Description - for SEO purposes

  • Published - Only available for Shop

No two pages are alike — adding new pages from NetlifyCMS is disabled.

Everything Else

While most content can be edited via NetlifyCMS there are a few bits tucked away in the site configuration file, namely the description, which you can override by editing the index.md file at Pages > Home.

Other defaults, such as site title and author are also stored in _config.yml.
